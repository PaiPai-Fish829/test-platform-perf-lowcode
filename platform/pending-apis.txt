Locust 压测监控平台 — 待接入接口清单
=====================================
生成时间：2026-05-29
Locust 版本参考：2.44.0
API 基址：{locustApiBase}（开发环境默认 /locust-api 代理至 VITE_LOCUST_URL）

一、已接入的 Locust 原生接口
----------------------------
[√] GET  /stats/requests          — KPI 卡片、API 统计表、失败明细、图表数据源（轮询 2s）
[√] GET  /exceptions              — 异常统计表
[√] GET  /logs                    — 底部日志预览（master 最近 5 条）
[√] POST /swarm                   — 测试任务页「立即执行」（user_count、spawn_rate、run_time）
[√] GET  /stop                    — 顶部「停止运行」按钮
[√] GET  /stats/requests/csv      — 导出：请求统计 CSV
[√] GET  /stats/failures/csv      — 导出：失败明细 CSV
[√] GET  /exceptions/csv          — 导出：异常 CSV
[√] GET  /stats/report?download=1 — 导出：HTML 报告

二、Locust 原生接口 — 尚未接入
------------------------------
[ ] POST /swarm（完整参数）
    - host：目标 Host 覆盖
    - shape_class：LoadTestShape 选择（阶梯/保持等）
    - user_classes：UserClass 多选（需 --class-picker）
    - profile：运行 profile
    - 其他 parsed_options 透传字段
    说明：当前仅传 user_count、spawn_rate、run_time，策略模板未映射到 shape_class

[ ] GET  /stats/reset
    说明：重置统计数据，可在「新任务开始前」或工具栏提供按钮

[ ] GET  /stats/requests_full_history/csv
    说明：需 Locust 启动时启用 stats_history_enabled；用于完整历史曲线导出

[ ] GET  /tasks
    返回：per_class / total 任务执行比例
    说明：可用于「场景/task 占比」可视化，替代 mock 场景成功率

[ ] GET  /worker-count
    说明：分布式模式下展示 Worker 数量（Master 节点）

[ ] GET  /stats/requests 中的 workers 字段
    说明：Master 模式下 Worker 列表（id、state、user_count、cpu、memory）
    建议：在「分布式执行」高级选项或独立面板展示

[ ] POST /user
    说明：运行时动态修改 User 类属性（Locust 高级用法）

[ ] GET  /login、Web 认证
    说明：若 Locust 启用 --web-login，前端需增加登录页与 Cookie 会话

[ ] GET  /metrics（本项目扩展）
    说明：common/metrics.py 暴露的 Prometheus 指标；可与 Grafana Tab 联动

三、平台自有业务 — 需新建后端 API
--------------------------------
以下功能在 AI 生成的 UI 中存在，但 Locust 不提供对应接口，需框架侧自行实现：

[ ] 测试场景库
    - 场景 CRUD、分组、标签、历史成功率
    - 对应 UI：「选择测试场景」列表、搜索、标签筛选
    - 建议：读取 scenarios/ 目录或 MySQL 场景表

[ ] 测试策略库
    - 策略模板 CRUD（阶梯增长、波浪形、突发峰值等）
    - 对应 UI：「管理策略库」、策略预览图
    - 建议：映射到 shapes/ 下 LoadTestShape 配置 + YAML 存储

[ ] 任务草稿 / 任务历史
    - 保存草稿、任务列表、定时任务调度
    - 对应 UI：「保存草稿」、任务名称、定时执行
    - 建议：MySQL 任务表 + 可选 Celery/APScheduler

[ ] 定时执行
    - 对应 UI：「定时执行」+ DatePicker
    - Locust 无内置定时；需平台调度器触发 POST /swarm

[ ] 思考时间 / wait_time
    - 对应 UI：高级选项「思考时间设置」
    - 需写入 locustfile User 类或场景配置，非 Locust Web API

[ ] CSV 参数化路径 / 数据分配策略
    - 对应 UI：「数据参数化」
    - 框架已有 data/users.csv + common/data_loader.py，需配置 API 读写

[ ] 分布式 Worker 管理
    - 对应 UI：Worker 节点数、Master 地址、期望 Worker 数
    - Locust 仅提供 worker 状态查询，Worker 进程启停需运维/脚本层

[ ] 结果输出路径 / 自动报告
    - 对应 UI：「自动保存测试报告」「输出 CSV 路径」
    - 建议：对接 scripts/run.py stress 无头模式 + reports/ 目录

[ ] WebSocket 实时推送
    - 对应 UI：「实时推送数据到 WebSocket」
    - Locust Web UI 使用轮询；若需 WS 需自建推送服务

[ ] Prometheus 导出开关
    - 对应 UI：「导出 Prometheus 格式」
    - 项目已固定暴露 /metrics；此处可做 Grafana 嵌入 Tab（VITE_GRAFANA_URL）

[ ] 测试报告模块
    - 对应 UI：「测试报告」菜单
    - 建议：聚合 /stats/report、历史 CSV、reports/ 目录文件列表

[ ] 脚本库 / 策略库菜单
    - 对应 UI：「测试资源」子菜单
    - 建议：浏览 tasks/、scenarios/、shapes/ 源码树

四、Grafana / 监控栈（可选）

[ ] Grafana 仪表盘嵌入
    - 环境变量：VITE_GRAFANA_URL

五、前端技术说明
----------------
- 原 zip 为 Vue + Element Plus + VTJ 低代码产物，已移植为 React + Ant Design + ECharts
- 图表历史：客户端轮询 /stats/requests 累积最近 60 点（与 Locust 原生 UI 类似）
- 开发跨域：vite.config.ts 配置 /locust-api 代理至 VITE_LOCUST_URL
